# Maneater

Go binary in `cmd/maneater`. Pure CLI for building and querying a semantic
search index over Unix man pages. Man page rendering, section reading, and Go
documentation are provided by native moxin tool configs (`moxins/man/`,
`moxins/gordo/`) rather than by maneater directly.

## Subcommands

- `maneater index` -- build/rebuild search index with progress output
- `maneater search <query> [--top-k N]` -- semantic man page search

## Search Architecture

CGo bindings to nixpkgs' libllama (`internal/embedding/llama.go`) embed man
page synopses and tldr pages using nomic-embed-text-v1.5. Key details:

- Must use `llama_batch_init` (not `batch_get_one`) with manual seq_id/logits
  setup, otherwise embeddings are all zeros
- Raw embeddings need L2 normalization (unlike llama-server's `/embedding`
  endpoint)
- Index deduplicates by page name, keeping the highest cosine similarity score
- Cached at `$XDG_CACHE_HOME/maneater/man-index/` (regenerable via
  `maneater index`)
- Model path from `MANEATER_MODEL_PATH` env var (set by nix wrapper)

## Manpath Discovery

Maneater automatically probes the current working directory for `man/`,
`doc/man/`, and `share/man/` subdirectories, prepending any that exist to the
system manpath. This makes project-bundled man pages discoverable without manual
`MANPATH` configuration. Controlled via `[manpath]` in `maneater.toml` —
`include` adds extra dirs, `no-auto = true` disables heuristic probing.

## Maneater.toml Hierarchy

Config files loaded and merged in order:

1.  `~/.config/maneater/maneater.toml` (global, falls back to `models.toml`)
2.  Each parent directory between `$HOME` and `$CWD`
3.  `./maneater.toml` (project-local)

Models are merged by name (overlay wins per key). The `config_tommy.go` file is
generated by `//go:generate tommy generate`.

## Nix Build

`maneater-unwrapped` uses `CGO_ENABLED = "1"` with `pkg-config` and `llama-cpp`
in buildInputs. Wrapped binary adds mandoc, pandoc, tldr to PATH.

## Testing

- Embedding tests in `internal/embedding/` require `MANPAGE_MODEL_PATH` env var
  pointing to the nomic GGUF model; they are skipped otherwise
- `search_quality_test.go` documents expected ranking behavior and known
  limitations --- update these when changing the embedding pipeline
